# H100

NVIDIA's Hopper architecture GPU, the most powerful GPU for deep learning as of 2023. The H100 SXM is the flagship data center GPU.

## Specifications

| Spec | H100 SXM | H100 PCIe |
|------|----------|-----------|
| Architecture | Hopper | Hopper |
| Tensor Cores | 4th gen | 4th gen |
| FP8 TFLOPS | 3,958 | ~2,000 |
| FP16 TFLOPS | ~1,979 | ~1,513 |
| Memory | 80 GB HBM3 | 80 GB HBM2e |
| Memory Bandwidth | 3.35 TB/s | 2.0 TB/s |
| TDP | 700W | 350W |

## Key Features

### Tensor Memory Accelerator (TMA)
- Hardware unit for computing memory read/write indices
- Frees GPU threads to focus on computation
- Enables fully asynchronous memory transfers
- Reduces effective memory latency by ~15%

### FP8 Support (First Consumer/Data Center GPU)
- 8-bit floating point for both inputs and compute
- 2x faster data loading than FP16
- 2x more data fits in caches
- Most stable 8-bit format (vs INT8)

### Largest Caches
- Largest L2 cache in any NVIDIA GPU
- Enables more data reuse from cache
- Better for large models (LLaMA, GPT-4 class)

### NVLink 3.0
- Faster multi-GPU interconnect
- 900 GB/s bidirectional bandwidth
- Better scaling for multi-GPU training

## Performance

- **Baseline**: 1.0x (used for all other GPU comparisons)
- **vs A100**: ~2-3x faster for transformers
- **vs RTX 4090**: ~3x faster (8-bit), higher for 16-bit
- **vs V100**: ~6-8x faster

## Use Cases

- Large-scale LLM training (GPT-4, Claude, Gemini class)
- Data center inference serving
- Research requiring maximum performance
- Multi-GPU training clusters

## Variants

### H100 SXM
- Higher bandwidth (HBM3)
- Higher power (700W)
- Requires NVIDIA DGX systems or compatible servers
- Best for multi-GPU configurations

### H100 PCIe
- Lower bandwidth (HBM2e)
- Lower power (350W)
- Fits in standard PCIe servers
- Good for single-GPU or small clusters

## Comparison to Consumer GPUs

| Feature | H100 | RTX 4090 |
|---------|------|----------|
| FP8 Tensor Cores | ✓ | ✓ |
| TMA Unit | ✓ | ✗ |

| VRAM | 80 GB | 24 GB |

**Note**: H100 has ~3x better performance per Tensor Core unit, not just from more cores.
