# A100

NVIDIA's Ampere architecture data center GPU, the workhorse for enterprise deep learning from 2020-2023. Succeeded by H100 but still widely deployed.

## Specifications

| Spec | A100 SXM | A100 PCIe |
|------|----------|-----------|
| Architecture | Ampere | Ampere |
| Tensor Cores | 3rd gen | 3rd gen |
| FP16 TFLOPS | 312 (sparse) | 312 (sparse) |
| Memory | 40/80 GB HBM2e | 40/80 GB HBM2e |
| Memory Bandwidth | 2,055 GB/s | 1,935 GB/s |
| TDP | 400W | 250W |

## Key Features

### Third-Generation Tensor Cores
- Support for TF32, BF16, FP16, INT8, INT4
- Structured sparsity for 2x throughput on sparse models
- Multi-instance GPU (MIG) partitioning

### Structured Sparsity (First Generation)
- Fine-grained automatic sparse matrix multiplication
- Weight matrices: 2 of every 4 elements must be zero
- Compressed to half size → 2x effective throughput
- Works with algorithms like RigL, Sparse Momentum

### Multi-Instance GPU (MIG)
- Partition A100 into up to 7 isolated instances
- Each instance has dedicated memory and compute
- Useful for serving multiple models simultaneously
- Ensures Quality of Service (QoS) guarantees

## Performance vs Previous Generation

From NVIDIA benchmarks (corrected for batch size, 1 GPU):

| Workload | A100 vs V100 |
|----------|--------------|
| SE-ResNeXt101 | 1.43x faster |
| Masked R-CNN | 1.47x faster |
| Transformer (12-layer MT) | **1.70x faster** |

For transformers, practical speedup matches theoretical estimate (~1.73x based on memory bandwidth ratio).

## Memory Bandwidth Advantage

- V100: 900 GB/s
- A100: 2,055 GB/s
- Speedup estimate: 2,055/900 = 2.28x theoretical, 1.70x practical

Gap explained by:
- Smaller tensor dimensions in some layers
- Overhead from img2col, FFT operations
- Final layers often too small to saturate GPU

## Variants

### A100 40GB
- Original release
- 40 GB HBM2e
- Lower cost option

### A100 80GB
- Double memory capacity
- HBM2e (not HBM3)
- Essential for large models (GPT-3 class and above)

### A100 PCIe vs SXM
- SXM: Higher bandwidth, used in DGX systems
- PCIe: More flexible, fits standard servers

## Comparison to H100

| Feature | A100 | H100 |
|---------|------|------|
| Architecture | Ampere | Hopper |
| Tensor Core Gen | 3rd | 4th |
| FP8 Support | ✗ | ✓ |
| TMA Unit | ✗ | ✓ |
| Memory Bandwidth | 2 TB/s | 3.35 TB/s |
| Max VRAM | 80 GB | 80 GB |
| Relative Performance | 1.0x | 2-3x |

**Verdict**: H100 is 2-3x faster, but A100 at lower price point remains good value.

## Comparison to Consumer GPUs

| Feature | A100 | RTX 3090 |
|---------|------|----------|
| Tensor Cores | ✓ (dedicated) | ✓ (consumer) |
| NVLink | ✓ | ✗ |
| VRAM | 40/80 GB | 24 GB |
| PCIe 4.0 | ✓ | ✓ |
| TDP | 250-400W | 350W |
| Price | ~$10,000+ | ~$1,500 |

A100 has slightly better Tensor Core utilization than consumer Ampere (RTX 30 series) due to not being deliberately degraded.
