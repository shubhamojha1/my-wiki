# CS 179: Synchronization and ILP - Lecture 5

## Course Details

- **Institution**: Caltech
- **Course**: CS 179: GPU Programming
- **Lecture**: 5

## Summary

Lecture 5 covers synchronization mechanisms in CUDA, floating-point precision issues, atomic operations, instruction-level parallelism (ILP), warp scheduling, and occupancy. Key insight: GPUs require different synchronization approaches than CPUs due to hardware architecture.

## Key Topics

### Synchronization

- `__syncthreads()`: Block-level barrier synchronization
- Deadlock prevention
- GPU-specific patterns vs CPU locks/semaphores

### Floating Point Precision

- Addition is NOT associative
- Numerical stability matters
- Strategy: Add smallest to largest
- Double vs single precision trade-offs

### Atomic Operations

- Built-in CUDA atomics: `atomicAdd`, `atomicCAS`, etc.
- Compare-and-swap implementation
- Performance cost — use sparingly

### Performance Optimization

- **ILP**: Reorder independent instructions
- **Warp Scheduler**: Hides latency via fast context switching
- **Occupancy**: Active warps / max warps ratio

## H200/H100 Specifications

| Resource | H200 | H100 |

| Max threads/block | 1024 (32 warps) | 1024 (32 warps) |
| Registers/SM | 64K 32-bit | 64K 32-bit |
| Max shared memory/SM | 228KB | 228KB |

## Occupancy Examples (H100)

**100% Occupancy**: 2 blocks × 1024 threads, 32 registers/thread, 114KB shared/block

**50% Occupancy**: 1 block × 1024 threads, 64 registers/thread, 228KB shared/block
