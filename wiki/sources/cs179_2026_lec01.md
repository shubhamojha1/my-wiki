## Summary

Lecture 1 introduces GPU programming fundamentals, contrasting CPU and GPU architectures and establishing motivation for parallel computing. The course uses CUDA (NVIDIA's parallel computing platform) with C++.

## Key Topics

1. **GPU Computing Motivation** — GPUs excel at parallelizable problems with thousands of simultaneous threads vs CPU's 8 cores
2. **GPGPU Evolution** — From fixed-function pipelines → shaders (GLSL) → CUDA/OpenCL
3. **GPU Memory Workflow** — Host(CPU) allocation → copy to GPU → kernel execution → copy back
4. **Thread Indexing** — Unique thread IDs via block ID and thread ID within block

## CPU vs GPU Architecture

| Aspect            | CPU                              | GPU                           |
| ----------------- | -------------------------------- | ----------------------------- |
| Cores             | 8 or fewer, powerful             | Thousands of simple cores     |
| Paradigm          | Sequential operations            | Massively parallel            |
| Thread count      | ~8 threads                       | ~20,000 threads               |
| Context switching | High penalty                     | Minimal overhead              |
| Best for          | General-purpose, branching logic | Data-parallel, SIMD workloads |

## GPU Computing Workflow

1. Setup inputs on host (CPU-accessible memory)
2. Allocate memory on GPU for inputs and outputs
3. Copy inputs from host to GPU (slow)
4. Launch kernel (parallel function execution)
5. Copy output from GPU to host (slow)

Note: Copying can be asynchronous; unified memory management is available.
