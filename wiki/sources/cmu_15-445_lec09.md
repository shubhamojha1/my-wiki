## Summary

Lecture 9 covers advanced index topics: more B+Tree details, specialized indexes (inverted, vector), and index optimization strategies.

## More B+Tree Details

- **Non-unique indexes**: Store duplicate keys or RID lists
- **Key compression**: Compress prefix keys in internal nodes
- **Bulk loading**: Build index by sorting then sequential inserts
- **Tombstones**: Mark deleted keys, ignore during search

## Inverted Index

- Maps values to document/record IDs
- Used for **full-text search**
- Like a book index: term → pages containing term

## Vector Index

- Used for **vector search** (embeddings)
- Enables nearest-neighbor queries
- Techniques: HNSW, IVF, PQ
- Used in [[Vector Database]]

## Index Optimizations

### Partial Index
- Index subset with WHERE clause
- Smaller, faster to maintain
- Example: Index recent orders only

### Index Include Columns
- Non-key columns stored in leaf
- Covering index without affecting sort order
- PostgreSQL: `INCLUDE (col1, col2)`

### Index-Only Scan
- Query satisfied entirely from index
- No table access needed
