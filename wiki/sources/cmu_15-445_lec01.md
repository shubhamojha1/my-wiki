## Summary

Lecture 1 introduces database systems, covering the distinction between databases and DBMS, problems with flat-file storage, and the relational model. It then covers relational algebra as the foundation for query languages like SQL.

## Database vs DBMS

- **Database**: Organized collection of interrelated data modeling real-world entities
- **DBMS** (Database Management System): Software that manages a database (MySQL, Oracle, PostgreSQL, MongoDB)

## Flat File Strawman Problems

Storing data in CSV files creates issues:
- **Data Integrity**: No enforcement of relationships, invalid values, orphan records
- **Implementation**: No indexing, portability issues, no concurrency control
- **Durability**: No crash recovery, replication challenges

## Relational Model

Proposed by Ted Codd at IBM (1969). Three components:
- **Structure**: Relations and attributes independent of physical representation
- **Integrity**: Constraints that data must satisfy
- **Manipulation**: Declarative API for querying and modifying data

Key abstractions: [[Relational Model]], [[Primary Key]], [[Foreign Key]], Constraints

## Data Manipulation Languages

- **Procedural**: Specify execution strategy (how)
- **Non-Procedural/Declarative**: Specify desired result (what)

SQL is the dominant declarative language. See [[SQL]].

## Relational Algebra

Fundamental operations for retrieving and manipulating tuples:
- **Select (σ)**: Filter tuples by predicate
- **Projection (π)**: Select specific attributes
- **Union (∪)**: Combine relations
- **Intersection (∩)**: Common tuples
- **Difference (−)**: Tuples in one but not other
- **Product (×)**: Cartesian product
- **Join (▷◱)**: Combine based on shared attributes

## Query Optimization

The DBMS decides the optimal execution plan. Declarative queries allow the system to optimize without user involvement. This is the power of SQL.

## Other Data Models

- **Document Model**: JSON-based, hierarchical (MongoDB)
- **Vector Model**: For embeddings and nearest-neighbor search (see [[Vector Database]])
- Key/Value, Graph, Wide-Column
