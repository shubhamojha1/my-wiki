## Summary

Lecture 7 covers hash tables as data structures used extensively in DBMS for internal data structures (buffer pool, catalog) and as index structures. It covers static hashing (linear probe, Robin Hood) and dynamic hashing (extendible, linear).

## Hash Table Basics

- **Hash function**: Maps key → array index
- **Collision**: When two keys hash to same index
- **Hash tables provide O(1) average-case lookup**

## Static Hashing

### Linear Probe Hashing
- Single fixed-size table of slots
- Collision: linearly search for next free slot
- **Robin Hood variant**: Swap items to minimize probe distance

### Chained Hashing
- Each slot has linked list (bucket) of entries
- Collision: add to bucket's linked list
- Simple but pointer-heavy

## Dynamic Hashing

### Extendible Hashing
- Uses directory pointing to buckets
- Split buckets incrementally when overflow occurs
- Directory doubles when needed
- Local vs global depth

### Linear Hashing
- Maintains split pointer
- Split any overflowing bucket, not just the one at pointer
- Pointer advances through buckets
- Eventually reshapes hash function

## Database Considerations

- **Non-unique keys**: Store list of RIDs or count
- **Fixed-length entries**: Easier to hash and store
- **Resize cost**: Static requires full rehash
