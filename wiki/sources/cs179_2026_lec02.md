## Summary

Lecture 2 dives into GPU hardware internals, contrasting SIMD and SIMT execution models, and explaining the thread hierarchy that underlies CUDA programming.

## Thread Organization Hierarchy

```
Grid
 └── Block 0, Block 1, ... Block N
      └── Warp 0 (32 threads)
      └── Warp 1 (32 threads)
      └── ...
           └── Thread (smallest unit)
```

- **Thread**: Smallest independent unit of execution
- **Warp**: Group of 32 threads executed in lockstep
- **Block**: Coordinated group of warps (1-1024 threads)
- **Grid**: All blocks for a kernel launch

## dim3 for Multidimensional Threading

CUDA provides `dim3` for defining grid and block dimensions:

```cuda
dim3 grid(256);           // 256 blocks, 1D
dim3 block(512, 512);     // 512x512 threads, 2D
foo<<<grid, block>>>(...);
```

Useful for linear algebra, physical modelling, and multidimensional arrays.

## Hardware Limits

| Parameter | Typical Limit |
|-----------|---------------|
| Max threads/block | 512 or 1024 |
| Max blocks/grid | 65535 |
| Threads/warp | 32 (fixed) |
| SMs per GPU | ~100 |

## Key Concepts Introduced

1. **SIMD**: Single Instruction, Multiple Data — vector registers
2. **SIMT**: Single Instruction, Multiple Threads — CUDA's model
3. **Streaming Multiprocessor (SM)**: ~100 per GPU, runs 128+ threads
4. **Warp Divergence**: Threads in same warp taking different paths
