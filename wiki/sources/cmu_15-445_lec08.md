## Summary

Lecture 8 introduces B+Trees—the dominant index structure in database systems. It covers B+Tree properties, node structure, operations (insert/delete), and composite indexes.

## B+Tree Properties

- **Self-balancing**: All leaf nodes at same depth
- **Sorted**: Keys in order for range scans
- **Fanout**: High branching factor (hundreds)
- **Height**: Typically 2-4 levels for millions of rows
- **Minimum occupancy**: Nodes at least half-full

## B+Tree Node Structure

- **Inner nodes**: Keys + child pointers (no data)
- **Leaf nodes**: Keys + values + next pointer
- Values: Either actual data (IOT) or row pointers

## B+Tree vs B-Tree

- **B-Tree**: Data in internal nodes and leaves
- **B+Tree**: Data only in leaves, internal nodes only guide search

## Operations

### Search
- Traverse from root to leaf
- O(log N) I/Os

### Insert
- Find leaf, insert key/value
- Split if node full (propagate up)

### Delete
- Find leaf, remove key
- Merge/coalesce if underfull

### Range Scan
- Find start key, follow leaf linked list

## Composite Index

- **Multiple columns**: Key = (col1, col2, ...)
- **Prefix matching**: Can use if query provides leftmost columns
- Supported: `(a=1 AND b=2)`, `(a=1)`
- Not supported: `(b=2)` alone
