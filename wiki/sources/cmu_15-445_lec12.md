## Summary

Lecture 12 covers algorithms for computing joins—the most expensive operation in query processing. It covers nested loop, sort-merge, and hash join algorithms.

## Join Types

- **Inner join**: Matching tuples from both tables
- **Outer join**: Include unmatched rows from one/both tables
- **Equijoin**: Join on equality predicate
- Focus: Inner equijoin algorithms

## Nested Loop Join

### Naive Nested Loop
- For each tuple in outer, scan all inner
- O(N × M) comparisons
- Put smaller table outer

### Block Nested Loop
- Load block of outer into memory
- Scan inner for each block
- Better cache utilization

### Index Nested Loop
- Use index on inner table for lookup
- Fast if index is selective

## Sort-Merge Join

1. **Sort**: Sort both tables on join key
2. **Merge**: Scan sorted tables, find matches
- O(N log N + M log N) if sorted
- Good for pre-sorted data or ranges

## Hash Join

### Basic Hash Join
1. **Build**: Hash outer table into hash table
2. **Probe**: Scan inner table, probe hash table
- O(N + M) if fits in memory
- Use smaller table as build

### Grace Hash Join (Partitioned)
1. Partition both tables on join key
2. For each partition pair, do basic hash join
- Handles when hash table doesn't fit in memory
- Recursive partitioning if needed

## Comparison

| Algorithm | Best For | Complexity |
|-----------|----------|------------|
| Nested Loop | Small tables, indexed | O(N×M) |
| Sort-Merge | Pre-sorted, ranges | O(N log N) |
| Hash Join | Equijoin, large tables | O(N+M) |
