## Summary

Lecture 11 introduces query processing—how the DBMS executes SQL queries. It covers the query plan, execution models (Volcano/Iterator), and access methods.

## Query Processing Overview

1. **Parser**: SQL → AST (abstract syntax tree)
2. **Binder**: AST → Logical plan (relational algebra)
3. **Optimizer**: Logical plan → Physical plan (execution strategy)
4. **Executor**: Physical plan → Results

## Query Plan

- **Tree of operators**: Data flows from leaves to root
- **Leaves**: Access methods (table scans, index scans)
- **Internal nodes**: Operators (joins, filters, aggregations)
- **Root**: Output to user

## Execution Models

### Volcano Model (Iterator)
- Each operator has `Next()` method
- Pull-based: parent calls child for tuples
- Simple, composable, but function call overhead

### Materialization
- Child computes all results, passes to parent
- Better for small results

### Vectorization
- Process batches of tuples (vectorized)
- Better CPU cache utilization, SIMD-friendly

## Access Methods

### Sequential Scan
- Read all pages, filter in memory
- When to use: small tables, no index, full table

### Index Scan
- Use index to find matching rows
- Point query, range scan
- Covering index possible

### Index Only Scan
- All needed columns in index
- No table access needed
