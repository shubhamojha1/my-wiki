# CMU 15-445: Modern SQL (Lecture 2)

## Course Details

- **Institution**: Carnegie Mellon University
- **Course**: 15-445/645 Database Systems (Spring 2026)
- **Instructor**: Andy Pavlo

## Summary

Lecture 2 covers modern SQL features beyond basic SELECT, including aggregates, grouping, window functions, CTEs, and nested queries.

## SQL History

- **Origins**: IBM System R project, 1970s → "SEQUEL" → "SQL"
- **Standards**: SQL-92 (minimum), SQL:1999, SQL:2003, SQL:2008, SQL:2011, SQL:2016, SQL:2023
- SQL is a **declarative** language: specify what, not how

## Relational Languages

- **DML**: SELECT, INSERT, UPDATE, DELETE
- **DDL**: Schema definitions
- **DCL**: Security and access control
- **Key difference**: Relational algebra uses sets; SQL uses bags (multisets with duplicates)

## Aggregates

Functions that reduce multiple rows to a single value:
- `AVG()`, `MIN()`, `MAX()`, `SUM()`, `COUNT()`
- Only in SELECT output list
- Support DISTINCT

## Grouping and Filtering

- **GROUP BY**: Partition tuples into groups
- **HAVING**: Filter groups (unlike WHERE which filters rows)
- **GROUPING SETS**: Multiple groupings in one query
- **FILTER clause**: Pre-aggregation filtering (SQL:2003)

## Advanced Queries

- **Nested Queries**: Subqueries in SELECT/FROM/WHERE
- **Lateral Joins**: Reference preceding query attributes
- **CTEs**: Common Table Expressions - named temporary result sets
- **Recursive CTEs**: Self-referencing CTEs, makes SQL Turing-complete

## Window Functions

Perform calculations across "windows" of related rows without collapsing them:
- `ROW_NUMBER()`, `RANK()`, `DENSE_RANK()`
- `OVER()` clause defines the window
- `PARTITION BY` groups rows

## Output Control

- **ORDER BY**: Impose sorting
- **LIMIT/OFFSET**: Pagination
