# CS 179: Matrix Transpose Optimization (Recitation 2)

## Overview

Lecture 6 covers optimizing matrix transpose on GPUs, applying concepts from previous lectures including [[Memory Coalescing]], [[Bank Conflicts]], and [[Tiling]]. The goal is to achieve memory-copy speeds for the transpose operation.

## Key Techniques

### Tiling
- Break matrix into 64x64 tiles
- Transpose each tile independently
- Each thread handles 4 values

### Warp Structure
- Warp = 32 adjacent threads
- Total of 32 warps for processing
- Each warp processes contiguous data

### Coalesced Memory Access
- Global memory requires coalesced reads/writes
- All 32 threads in warp should read/write adjacent floats
- 128 bytes total per transaction (32 threads × 4 bytes)

### Column-Major Storage
- Naive sequential reads are suboptimal due to column-major layout
- Must draw out access patterns to understand

### Shared Memory Intermediate
- Shared memory has different optimal access patterns than global memory
- Divided into 32 banks (striped layout)
- Every 128 bytes split evenly across 32 banks
- Optimal: each thread accesses different bank

### Bank Conflict Resolution
- Add 1 value of padding every 64 values
- Prevents serialization when multiple threads access same bank

### Optimal Transpose Algorithm
1. Load from global memory with coalesced access
2. Store to shared memory with padding (1 value per 64)
3. Read back from shared memory transposed
4. Store to global memory with coalesced access

### Loop Unrolling
```c
for (i = 0; i < 10; i++) {
    output[i] = a[i] + b[i];
}
```
- Reduces loop overhead
- Increases parallelism when iterations are independent
- May increase register usage
