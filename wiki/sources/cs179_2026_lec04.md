# CS 179: GPU Memory Systems - Lecture 4

## Course Details

- **Institution**: Caltech
- **Course**: CS 179: GPU Programming
- **Lecture**: 4

## Summary

Lecture 4 covers GPU memory systems in depth, contrasting latency with throughput, explaining the memory hierarchy from registers to global memory, and teaching optimization techniques like memory coalescing, avoiding bank conflicts, and maximizing computational intensity.

## Key Topics

### Memory Hierarchy

| Memory Type | Latency | Scope | Key Characteristics |
|-------------|---------|-------|-------------------|
| Registers | ~5ns | Thread | Fastest; thread-private |
| Shared Memory | ~5ns | Block | Fast; block-shared |
| L1 Cache | ~34 cycles | SM | Auto-managed |
| L2 Cache | ~200 cycles | GPU | Shared across SMs |
| L3 Cache | Slower than L2 | GPU | Larger but slower |
| Global Memory | ~300-600ns | Application | Persistent |

### Latency vs Throughput

**CPU**: Low latency (~100ns), low throughput
- Clock: 3 GHz (3 clocks/ns)
- Main memory latency: ~100+ ns
- Arithmetic latency: ~1+ ns

**GPU**: High latency (~300ns), high throughput
- Clock: 1 GHz (1 clock/ns)
- Global memory latency: ~300+ ns
- Arithmetic latency: ~10+ ns

**Key insight**: GPU is I/O limited. Need multiple FLOPs per memory load to get beyond ~2 TFLOPS.

### Memory Coalescing

Memory transactions are done per warp (32 threads) in 128-byte aligned cache lines. Coalesced access = optimal pattern where adjacent threads access adjacent memory.

**Misaligned/non-coalesced access** causes multiple cache line fetches = wasted bandwidth.

### Bank Conflicts

Shared memory has 32 banks. A bank conflict occurs when threads in a warp access different elements in the same bank, causing serial access.

**Stride impact**:
- Stride 1: Conflict-free
- Stride 2: 16× 2-way conflicts
- Stride 4: 8× 4-way conflicts
- Stride 32: 1× 32-way conflict (worst)

**Solution**: Padding (add 1 byte per row).

### Register Spilling

When registers are exhausted, stack variables spill to local memory (stored in global memory). This is 150x slower than register access.

**Cost**: Extra loads, stores, and dependent instructions.

### Computational Intensity

Ratio of compute operations to memory I/O (FLOPs/IO).

- Matrix multiplication: n³ / n² = n
- N-body simulation: n² / n = n

If >1, same data is used in multiple computations.

## Common Kernel Pattern

```
1. Copy from global memory to shared memory
2. __syncthreads()
3. Perform computation, store in shared memory
4. __syncthreads() (if needed)
5. Copy output from shared memory to global memory
```

## Memory Types Summary

| Type | Speed | Scope | Persistent |
|------|-------|-------|------------|
| Registers | Fastest | Thread | Thread lifetime |
| Local Memory | 150× slower | Thread | Thread lifetime |
| Shared Memory | Fast | Block | Block lifetime |
| Global Memory | Slowest | Application | Kernel launches |
| Constant Memory | Fast (cached) | Application | Host-set |
| Texture Memory | Fast | Application | Various |
