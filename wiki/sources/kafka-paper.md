# Kafka: A Distributed Messaging System for Log Processing

## Paper Details

- **Authors**: Jay Kreps, Jun Rao, Neha Narkhede (LinkedIn)
- **Published**: 2011
- **Venue**: Not publicly known

## Summary

Kafka is a distributed publish-subscribe messaging system designed to replace traditional log aggregation infrastructure. It provides a unified platform for handling all real-time feeds of event data generated by applications and systems.

The key innovation is building Kafka on a distributed, partitioned, and replicated **commit log** service that guarantees fault tolerance and durability while achieving high throughput.

## Key Contributions

1. **Commit Log Foundation** — Kafka stores messages in a persistent, ordered commit log rather than traditional message queue semantics
2. **Producer/Consumer Model** — Decoupled producers push data to topics; consumers pull data at their own pace
3. **Partitioned Topics** — Topics are partitioned across brokers for horizontal scalability
4. **Replication** — Configurable replication factor provides fault tolerance
5. **Retention** — Messages are retained by time or size, not deleted on consumption

## Architecture Highlights

- **Brokers**: Stateless storage nodes that form the Kafka cluster
- **Topics**: Logical channels for organizing messages
- **Partitions**: Subdivisions of topics for parallelism
- **Consumers**: Consumer groups enable parallel consumption with message ordering guarantees

## Legacy

This paper established the core design patterns that made Kafka the backbone of real-time data pipelines at scale. Later work extended Kafka with stream processing capabilities (Kafka Streams), SQL interfaces (ksqlDB), and integration with Hadoop ecosystems.
