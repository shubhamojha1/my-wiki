## Summary

Lecture 5 covers advanced storage organization: slotted pages for variable-length data, index-organized tables (IOT) that store data in B+Trees, and log-structured storage for write-heavy workloads.

## Slotted Page Storage

- Tuples stored in pages with a **slot array** at the end
- Slot array maps slot number → tuple offset
- Allows **variable-length tuples** and **tuple movement**
- Header tracks number of slots and free space start

## Index-Organized Storage

- **IOT (Index-Organized Table)**: Primary key is the B+Tree index
- Data pages are the leaf nodes of the B+Tree
- Good for **point queries** and **range scans** on primary key
- Used in: Oracle, MySQL InnoDB, SQL Server

## Log-Structured Storage

- Instead of updating pages in place, **append only log**
- Write records: INSERT, DELETE, UPDATE
- Periodically **compact** logs into new data files
- Good for **write-heavy** workloads
- Read requires replaying log or maintaining index

## Comparison

| Approach | Best For | Read Cost | Write Cost |
|----------|----------|-----------|------------|
| Heap + Slotted | General purpose | Medium | Medium |
| Index-Organized | PK lookups, range scans | Low (on PK) | Higher (maintain order) |
| Log-Structured | Write-heavy, append-only | Higher (replay) | Low (append only) |
