# BF16 (BrainFloat16)

BrainFloat16. A 16-bit floating point format with the same range as FP32 but reduced precision. Designed specifically for deep learning training.

## Format Specification

| Format | Sign | Exponent | Mantissa | Total Bits |
|--------|------|----------|----------|------------|
| FP32 | 1 | 8 | 23 | 32 |
| FP16 | 1 | 5 | 10 | 16 |
| **BF16** | 1 | 8 | 7 | 16 |

### Key Difference from FP16

**FP16 range**: [-65,504, 65,504]
**BF16 range**: [-3×10^38, 3×10^38] (same as FP32)

BF16 uses 3 extra exponent bits compared to FP16, giving it the same dynamic range as FP32.

### Trade-off

- **FP16**: More precision (10 mantissa bits) but limited range
- **BF16**: Less precision (7 mantissa bits) but unlimited range

For deep learning, **gradient precision is less important than range**. Gradients can be very small or very large; having sufficient range prevents gradient explosion.

## Why BF16 Matters

### The FP16 Problem

FP16 training requires **loss scaling**:
1. Multiply loss by a scaling factor before backprop
2. Prevents gradients from exceeding FP16 range
3. Un-scale gradients after backprop
4. Requires careful tuning of scaling factor

### BF16 Solution

With BF16:
- Same range as FP32 → gradients won't explode
- No loss scaling needed
- Drop-in replacement for FP16
- Same speed as FP16

**Result**: More stable training without extra code complexity.

## Usage in Practice

### PyTorch Example

```python
# Enable BF16
model = model.half()  # Not BF16, this is FP16

# For BF16, use:
model = model.to(torch.bfloat16)

# Or via autocast
with torch.autocast(device_type='cuda', dtype=torch.bfloat16):
    output = model(input)
```

### Automatic Mixed Precision (AMP)

```python
scaler = torch.cuda.amp.GradScaler()

for data, target in dataloader:
    optimizer.zero_grad()
    
    with torch.autocast(device_type='cuda', dtype=torch.bfloat16):
        output = model(data)
        loss = loss_fn(output, target)
    
    scaler.scale(loss).backward()
    scaler.step(optimizer)
    scaler.update()
```

## Comparison with Other Formats

| Format | Range | Precision | Speed | Use Case |
|--------|-------|-----------|-------|----------|
| FP32 | Full | Highest | Baseline | Gradient computation |
| TF32 | Full | High | ~FP16 | Training (easy switch) |
| BF16 | Full | Medium | ~FP16 | Training (stable) |
| FP16 | Limited | High | Fastest | Inference, with loss scaling |
| FP8 | Limited | Low | Fastest | Inference, mixed precision |

## Recommendations

1. **Use BF16 for training** — Stability of FP32 with speed of FP16
2. **No loss scaling needed** — Simpler than FP16 training
3. **Use TF32 if lazy** — One-line change from FP32
4. **Keep master weights in FP32** — For gradient accumulation accuracy
